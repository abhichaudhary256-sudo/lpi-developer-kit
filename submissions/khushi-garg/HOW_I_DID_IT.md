# How I Did It – Khushi Garg

To complete Level 2 of the LPI Developer Kit, I followed a structured approach to set up the environment, verify tools, and test AI model outputs.
First, I cloned the LPI Developer Kit repository from GitHub to my local system. After cloning, I installed all the required dependencies using npm to ensure the project could run without errors.
Next, I executed the command `npm run test-client` to verify whether all the provided tools were working correctly. This step was important because it confirmed that the LPI server was properly set up and all tools were responding as expected. The output showed that all tools passed successfully, which indicated that the environment was correctly configured.
After verifying the tools, I set up a local AI model using Ollama. I selected the LLaMA3 model because it runs locally without requiring external APIs, making it efficient and easy to use. I ensured that the model was properly installed and running on my system.
Then, I tested the AI model by giving prompts related to digital twin concepts and the SMILE methodology. The model successfully generated responses explaining these concepts, which confirmed that both the model and the tools were functioning correctly together.
Finally, I documented all my findings, outputs, and learnings in the `level2.md` file. I included the test client results, AI model outputs, model selection reasoning, and my personal reflection based on the SMILE framework.
This entire process helped me understand how AI agents interact with tools, how local models can be used effectively, and how structured methodologies like SMILE can guide AI responses.
